Инструкция по настройке Devin

Подключите репозиторий к Devin, настройте баланс и начните работу с ИИ-ассистентом. Руководство проведёт по всем шагам — от авторизации до запуска первой сессии.

Необходимые данные Авторизация Баланс GitHub Запуск сессии Контроль баланса

★ Необходимые данные

  • GitHub-аккаунт — ваш личный, с доступом к нужному репозиторию.
  • Аккаунт Devin — приходит в боте после покупки в одном из трёх форматов:
    • Вариант 1. Логин----Пароль — прямой вход, разовый код не нужен.
    • Вариант 2. Логин----Пароль----Токен — вход через GitHub + одноразовый 2FA-код (генерируется из токена).
    • Вариант 3. email----пароль----client_id----refresh_token — вход по логину Devin + разовый код входа (Login Code) из письма.

Тип определяется по числу разделителей ----. В сообщении бота после покупки уже указано, какой именно у вас вариант и что делать.

1 Авторизация в Devin

Перейдите на app.devin.ai и выберите способ входа, соответствующий вашему варианту аккаунта.

Вариант 1 — Логин + Пароль

  • Введите логин и пароль в форме входа (поле находится в нижней части страницы).
  • Разовый код не требуется — вы сразу попадёте в аккаунт.

Вариант 2 — Логин + Пароль + 2FA-токен (GitHub)

  • На странице входа выберите вход через GitHub и войдите по выданным логину и паролю.
  • Когда GitHub запросит код двухфакторной аутентификации — сгенерируйте его из выданного токена.
Откройте генератор кодов, вставьте выданный токен — 6-значный код обновляется каждые 30 секунд. Открыть генератор 2FA

Скопируйте код и введите его при запросе 2FA. Код считается прямо в браузере — токен никуда не отправляется.

Вариант 3 — вход по логину Devin (разовый код из письма)

  • Введите выданный логин (email) и пароль в форме входа Devin.
  • Когда Devin запросит разовый код входа (Login Code), получите его из письма на странице «Почта».
Вставьте строку аккаунта email----пароль----client_id----refresh_token — страница покажет код входа из письма Devin. Открыть почту

Можно вставить сразу несколько аккаунтов (по одному в строке). Письма читаются напрямую через защищённое подключение — данные не сохраняются на сервере.

2 Настройка организации и баланса

  • После входа вы попадёте в список сессий (Devin Sessions).
  • В левом верхнем углу нажмите на имя пользователя → «Settings».
  • Откройте вкладку «Usage & Limits» и проверьте остаток баланса.
Убедитесь, что средств достаточно для запуска сессии — рекомендуется иметь от 5 $.

3 Подключение вашего GitHub-аккаунта

  • В настройках откройте вкладку «Connections» → «GitHub» → «Add connection».
  • Важно: выберите «Add to this organization», не создавайте новую организацию.
  • На странице GitHub войдите под своим личным аккаунтом (у которого есть доступ к репозиторию).
  • При первом подключении нажмите зелёную кнопку «Install» для установки приложения Devin.
Если приложение уже установлено, но возникла ошибка:
  • Нажмите «Connect» → откроется GitHub.
  • Нажмите «Uninstall» (удалите приложение).
  • Закройте страницу, снова нажмите «Connect» и затем «Install» — права доступа переустановятся.

После успешного подключения отобразится статус «GitHub Connected». Нажмите «Get started».

4 Запуск сессии и работа с агентом

  • Вы вернётесь на страницу списка сессий (Devin Sessions).
  • Выберите модель — рекомендуется Opus 4.5 или GPT-5.5 — и укажите репозиторий для работы.
Совет по экономии баланса:
  • Держите баланс не менее 5 $, чтобы агент выполнил задачу без остановки.
  • Останавливайте сессию вручную, когда агент завершил ключевые действия, чтобы не тратить средства впустую.

Пример запроса к агенту (формулировка для максимальной пользы):

«Создай подробную инструкцию для ИИ-разработчиков (AI instructions), обнови техническую спецификацию проекта и загрузи все исходные файлы в корень репозитория на GitHub».
